# نزدیکام (Nazdikam) — RTL Wireframes
## High-Fidelity Screen Blueprints · Mobile-First · Persian RTL

**Version:** 1.0 — 2026-06-11  
**Notation Key:**  
`[RTL]` = right side is the reading start  
`←` scroll = scroll direction is right-to-left  
`🔒` = locked / requires subscription  
`[API]` = value loaded dynamically from API (never hardcoded)  
`─────` = visual divider  

---

## RTL Layout Conventions Applied Throughout

| Element | RTL Placement |
|---|---|
| Primary text | Starts from the RIGHT edge |
| Section title | RIGHT side; "see all" link at LEFT |
| Back button / chevron | Points LEFT (←), placed top-LEFT |
| Primary CTA button | RIGHT-aligned or full-width |
| Search icon | Inside input on the RIGHT side |
| Clear/X button | Inside input on the LEFT side |
| Desktop sidebar | RIGHT side of page |
| Admin sidebar | RIGHT side of page |
| Step 1 in wizard | Rightmost step visually |
| Notification dot | Top-LEFT of avatar icon |
| Carousel scroll | Starts RIGHT, scrolls toward LEFT |
| Form labels | RIGHT-aligned above field |
| Field error text | RIGHT-aligned below field |
| Table actions column | LEFT-most column (reached last in RTL) |

## SCREEN 1 — Homepage (بازار) — `/`

### 1A · Visual Layout Description

The homepage is the primary discovery surface. It opens with a sticky header containing the logo (right), a full-width search bar, a province/city location chip, and login CTA or user avatar (left). Below the header, a full-width advertisement carousel auto-advances RTL. A horizontal category icon row follows. The page then renders a sequence of content rows — Featured Businesses, Featured Products, Featured Services, Discount Products, Discount Services, Installment Products, Installment Services, Videos, Provinces — each as a horizontally-scrolling RTL card row with a section header. On desktop, the header gains a secondary navigation bar with category dropdowns and location selectors. Content rows expand to 3-column grids instead of scrolling rows. A sticky bottom navigation bar is present on mobile only (4 or 5 tabs, RTL order: Profile at left, Home at right).

## SCREEN 2 — Search Results — `/search?q=...`

### 2A · Visual Layout Description

The search results page maintains the sticky header with the active search query pre-filled and editable. Directly below is a filter bar: a row of active filter chips (province, category, discount, installment) scrolling RTL, a "فیلترها" button that opens a bottom-sheet drawer, and a sort dropdown. A tab bar shows three tabs — Businesses / Products / Services — each with a result count badge. Results render in a single-column list on mobile (scrollable, infinite scroll). On desktop, results switch to 2-column (businesses) or 3-column (products/services) grids. The filter drawer on mobile is a full bottom-sheet with stacked filters. On desktop, filters appear as a sidebar panel on the right side.

## SCREEN 5 — Business Dashboard — `/dashboard/business/[id]`

### 5A · Visual Layout Description

The Business Dashboard is the owner's command center. The screen opens with a subscription status banner — color-coded (yellow = no subscription, orange = expiring soon, red = expired, green = active). Below, a business identity row shows the logo, business name, public page link, and verification status. A stats summary row shows 4 KPI cards (views today, views this week, products count, services count), but only if `can_view_page_analytics` is true; otherwise, the row is hidden. A 2×3 quick access grid of section cards follows, each showing an icon, label, item count, and a lock badge if the user lacks the required entitlement. An identification verification notice card shows current verification status. If `can_use_referral_system` is true, a referral card shows the invite code. On desktop, a right sidebar provides full section navigation with lock indicators.

## SCREEN 6 — Subscription Plans — `/dashboard/business/[id]/subscription/plans`

### 6A · Visual Layout Description

The plan selection page loads all plans dynamically from the API (`GET /api/v1/plans`). No plan names, feature sets, or price tiers are hardcoded. Each plan renders as a card with: plan name (from `plans.name`), price in Toman (Persian formatted), duration (computed from `duration_days`), a list of enabled feature flags (rendered with Persian label mappings), a list of usage limits (e.g., "تا ۵۰ محصول"), and a "انتخاب این پلن" CTA. Plans are sorted by `sort_order`. The current active plan (if renewing) is highlighted differently. A collapsible full-comparison table appears at the bottom. On mobile, plans stack vertically and scroll. On desktop, plans appear in a responsive grid (2 or 3 columns depending on count).

## SCREEN 7 — Analytics Dashboard — `/dashboard/business/[id]/analytics`

### 7A · Visual Layout Description

The analytics dashboard is fully gated behind `can_view_page_analytics`. When locked, the entire content area is blurred with a lock overlay and upgrade CTA. When unlocked, the page opens with a Jalali date range picker and preset buttons (7 / 30 / 90 days — limited by `analytics_history_days` from the plan snapshot). A KPI cards row shows 4 metrics: total page views, unique visitors, contact clicks, navigation clicks — each with a delta percentage vs. the previous period. A line/area chart shows visit trend over the selected period, with Jalali dates on the x-axis rendered RTL. Below the chart, tabs switch between: Overview / Products / Services / Videos — each revealing a performance table with sortable columns. All metric keys and breakdown keys come from the API response and are never hardcoded.

## SCREEN 8 — Admin Dashboard — `/admin`

### 8A · Visual Layout Description

The Admin Dashboard is desktop-first, with no bottom navigation bar. The layout has a persistent right sidebar (RTL) with full section navigation grouped by domain: Overview, Users, Businesses, Content (Products/Videos/Announcements), Subscriptions (Plans/Codes/History), Locations, Categories, Banners, Blog, Settings. The sidebar shows item counts and alert badges. The main content area opens on the Overview — a grid of platform KPI cards, a recent activity feed, a pending moderation queue summary, and charts for platform-level analytics. Quick action buttons provide shortcuts to the most common admin tasks. Each section is accessible via sidebar navigation. Admin tables are sortable, filterable, paginated (50 rows default), and support bulk selection with a checkbox column. The edit panel slides in from the left (RTL: from the right) as a side panel.
